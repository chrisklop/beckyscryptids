# Playbook v0.2 — UX Direction Pass

> Closes the "I don't know what to do" gap from v0.1.0 playtest. Adds AdVenture-Capitalist-style legibility (milestone meters, best-buy hint, bulk-buy controls, reveal placeholders, rare-event toasts, prestige progress strip) without breaking the existing dashboard feel.

**Out of scope:** audio (v0.3), Mebro reveal scaffolding (still Phase 4), Era 4–12 content, achievements page, tutorial overlay, "series" packs.

**Math foundation is untouched.** The reachability simulator and invariant tests from v0.1.1 must continue to pass.

---

## 1. Goal anchor philosophy

Two visual layers combine, neither alone is enough:

- **B (info-rich, the base):** every card shows its own milestone progress bar + "next ×N at M" hint. A thin prestige progress strip lives above the tab bar at all times.
- **C (best-buy highlight, the accent):** the Pecorella overtake metric runs in real time. The card identified as the optimal next purchase gets a 2px accent ring + "RECOMMENDED" pill in the top-right. C is gated by a Settings toggle (default ON).

The two layers compose: cards always show meters; the recommended card additionally gets the ring on top.

## 2. Per-card additions

`src/ui/components/GeneratorCard.vue` grows three new elements + an animation hook:

### 2.1 Milestone progress bar

A horizontal bar (3px tall) below the description line. Fills from 0% at the previous milestone to 100% at the next milestone:

```
progress = (owned - prev_milestone) / (next_milestone - prev_milestone)
```

If `owned < milestones[0]`, prev is 0 and next is `milestones[0]`. If `owned ≥ milestones[milestones.length - 1]`, the bar is hidden (no more milestones).

### 2.2 "Next ×N at M" hint label

A single-line caption directly below the bar:

```
×10 / next ×2 at 25
```

Format: `×<current_multiplier> / next ×<next_multiplier> at <next_milestone>`.

Hides when no milestones remain.

### 2.3 Owned count

Shown on the card meta line **for every card** (currently only non-click-driven cards show it). For click-driven Tier 1, this is what the user couldn't find — how many Sycophants they own.

### 2.4 Milestone-cross animation

When `owned` enters a milestone bucket (state went from `owned < M` to `owned >= M`):

- Card root flashes a warm tint over 400ms (`background` lerps from `surface` → `surface + 20% white` → `surface`).
- Meter does a "pop" reset: briefly expands to 100%, then snaps to 0%. (~150ms.)
- A small `+×N` label slides up from the rate indicator and fades over 800ms (transform: `translateY(0 → -16px)`, opacity `1 → 0`).

CSS handled. No JS animation libs.

### 2.5 "Recommended" ring (C overlay)

When `recommendedGenId.value === gen.id` and the user's `show_best_buy_hint` setting is true:

- 2px accent border (`--theme-accent`)
- "RECOMMENDED" pill (font-size 7px, accent background, era-themed letter spacing) absolutely positioned at top-right, -8px above the card

## 3. Global bulk-buy toolbar

A new component `src/ui/components/BulkBuyBar.vue`, mounted in `PlayTab.vue` between `<ResourceRow>` and the cards.

### 3.1 Visual

```
┌───────┬───────┬────────┬───────┐
│  ×1   │  ×10  │  ×100  │  Max  │
└───────┴───────┴────────┴───────┘
```

Segmented control. Selected tier is highlighted with accent background.

### 3.2 Progressive unlock

Locked tiers are NOT rendered (the bar grows wider as tiers unlock; no empty slots). Unlock states:

| Tier | Visible when |
|------|--------------|
| ×1   | always |
| ×10  | `state.prestige_count >= 1` |
| ×100 | `state.prestige_count >= 5` |
| Max  | `state.prestige_count >= 25` |

When a tier first unlocks, fire the bulk-buy-unlock toast (§5.3.4).

### 3.3 Behavior

- The selected multiplier persists in `state.bulk_buy_multiplier` (saved).
- When a tier is selected and a non-click-driven card is tapped, buy that many — using `computeBulkCost(base, growth, owned, n)` to compute total cost. If the player can't afford `n`, fall back to buying `floor(rumor / cost_of_next)` (greedy partial buy). Never error; just buy what's possible.
- `Max` resolves at click time to `floor((rumor + bulk_remainder) / current_growth_step)` — the number of buys that exhaust available Rumor. Closed-form via inverse of the bulk-cost formula.
- The cost shown on each affected card updates to reflect the current multiplier (`computeBulkCost` result, not just the per-unit cost).
- Click-driven Tier 1 is unaffected — bulk-buy applies to non-click-driven only.

### 3.4 New helper: bulk-affordable count

Add to `src/game/era-layer.ts`:

```ts
/** Largest N such that computeBulkCost(...) <= available, computed in closed form. */
export function maxAffordableBulk(
  base: number,
  growth: number,
  owned: number,
  available: number,
): number;
```

Derivation: invert the bulk-cost formula. With `S = base × growth^owned × (growth^n - 1) / (growth - 1) ≤ available`, solve for `n`:

```
n ≤ log( 1 + available × (growth - 1) / (base × growth^owned) ) / log(growth)
```

Floor it. Returns 0 if `available < computeCost(base, growth, owned)`.

Unit tested with at least 4 cases: zero affordable, exact-one, mid-quantity, very-large quantity.

## 5. Toast notification system

A new component `src/ui/components/ToastStack.vue`, mounted in `AppShell.vue` as a fixed overlay near the top of the safe area.

### 5.1 Visual

- Slide down 8px from the top, fade in over 200ms
- Auto-dismiss after 3000ms
- Stack vertically, max 3 visible (oldest drops out the top)
- Era-themed: parchment in Antiquity, blackletter banner in Era 2, broadsheet in Era 3
- Touch dismissable (tap to clear)

### 5.2 State + persistence

```ts
// in SaveState (see §7)
seen_toast_events: string[];  // event ids already fired

// toast manager
type ToastEvent = {
  id: string;          // unique key, deduped against seen_toast_events
  message: string;
  era_id?: string;     // for styling
  fired_at_ms: number;
};
```

A toast is fired iff its `id` is not already in `seen_toast_events`. After firing, the id is added.

### 5.3 The four trigger events

#### 5.3.1 First Sycophant auto-unlock
- Event id: `auto-unlock:${era_id}:${gen_id}` (e.g. `auto-unlock:antiquity:spread-rumor`)
- Fires when click-driven generator first crosses `owned >= auto_unlock_at`
- Message format: `"${operative_name} hired — idle Rumor flowing."` (e.g. `"Sycophant hired — idle Rumor flowing."`)
- Once per era (resets ID on prestige reset since the saved-events set persists across runs; the era_id in the id keeps it unique per era)

#### 5.3.2 Generator reveal
- Event id: `reveal:${era_id}:${gen_id}`
- Fires when a previously-locked generator's threshold is crossed
- Message: `"${display_name} appears."`

#### 5.3.3 Prestige available
- Event id: `prestige-ready:${era_id}`
- Fires when `projectedMI >= 1` first becomes true
- Message: era-themed via `currentCopy.prestige_ready_toast` (new field — default `"The threshold calls. Ascend whenever you're ready."`)

#### 5.3.4 Bulk-buy tier unlocks
- Event ids: `bulk-unlock:10`, `bulk-unlock:100`, `bulk-unlock:max`
- Fires when prestige_count threshold met (1 / 5 / 25)
- Messages: `"×10 buy unlocked."` / `"×100 buy unlocked."` / `"Max buy unlocked."`
- Once globally (across all eras), since bulk-buy is a meta-progression

### 7.1 Schema migration

Adds a `version: 2` constant to `CURRENT_SAVE_VERSION`. Migration tested with both fresh and pre-migrated save round-trips.

### 7.2 prestige_count wiring

## 9. Best-buy reactivity (the C piece)

Add a computed ref `recommendedGenId` in `src/ui/state.ts`:

```ts
export const recommendedGenId = computed<string | null>(() => {
  if (!state.show_best_buy_hint) return null;
  const visible = currentEra.value.generators.filter(g => state.lifetimeRumor >= g.reveal_at_lifetime);
  if (visible.length === 0) return null;
  return pickOptimalGenerator(visible, state, currentEra.value, carryoverMultiplier(state.memeticInheritance));
});
```

`pickOptimalGenerator` is a new pure function in `src/game/era-layer.ts`:

```ts
/** Pecorella overtake heuristic: returns the gen.id with the lowest cost/nps + cost/(nps+delta) score. */
export function pickOptimalGenerator(
  candidates: GeneratorTier[],
  state: { rumor: number; ownedByGenerator: Record<string, number>; },
  era: EraDefinition,
  globalMultiplier: number,
): string | null;
```

Unit tested with at least 3 scenarios: empty list, single candidate, multi-candidate with clear winner.

The computed ref reactively updates on every state change. Cheap (linear in 5–6 candidates).

## 12. Success criteria

After v0.2 ships:

- A new player crosses 10 Sycophants and sees a toast: `"Sycophant hired — idle Rumor flowing."`
- The player can articulate "I'm working toward Memetic Inheritance" — because the prestige strip is always visible
- Each card's milestone meter visibly fills, and when 25/50/100 owned is crossed, the card flashes warm + a `+×2` label rises and fades
- After first prestige, a `×10` button appears in the bulk-buy toolbar with a toast: `"×10 buy unlocked."`
- The "Recommended" pill visibly nudges the player toward the optimal next purchase in early Era 1
- Cost styling is consistent across all cards (no rogue bold/glow)
- All v0.1.1 reachability invariants still pass (math foundation untouched)
- All existing tests still pass
- `pnpm check:links` still reports all 18 codex source URLs reachable
